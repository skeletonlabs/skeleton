## Learn the Basics

If this is your first time using Floating UI, start here:

* [Floating UI Tutorial](https://floating-ui.com/docs/tutorial) - covers the basics of using native Javascript.
* [Floating UI React Tutorial](https://floating-ui.com/docs/react) - covers the basics of using the React wrapper.

### Imports

Floating UI React exposes all required Floating UI middleware, types, etc.

```ts
import { useFloating } from '@floating-ui/react';
```

### Making Elements Float

You'll need a dedicated [CSS class](https://floating-ui.com/docs/tutorial#making-the-tooltip-float) to apply common styles to all floating elements. Apply the following in your global stylesheet. We'll use the following class convention for all Skeleton examples.

```css
.floating {
	width: max-content;
	position: absolute;
	top: 0;
	left: 0;
}
```

### Z-Index Stacking

Please be aware that Floating UI does not take an opinionated stance on [z-index stacking](https://floating-ui.com/docs/misc#z-index-stacking). This means that if a floating element is occluded by another element in the z-axis, you must handle this directly.

### Middleware

Make sure you're familiar all available [Floating UI Middleware](https://floating-ui.com/docs/middleware), which provides modules for customizing the look and feel of popovers.

## Native Browser APIs

Skeleton will always favor native browser APIs over third-party libraries such as Floating UI Svelte. The following is a list of current and upcoming incoming APIs we will aim to support in the future, but are not quite standardized cross-browser yet.

* [Dialog Element](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dialog)
* [Popover API](https://developer.mozilla.org/en-US/docs/Web/API/Popover_API)
* [CSS Anchoring Position](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_anchor_positioning)

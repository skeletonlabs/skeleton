<p class="text-xl"> Themes are the keystone of Skeleton. Each theme's [CSS custom properties](https://developer.mozilla.org/en-US/docs/Web/CSS/--*) enable you to define core settings for your design system. Enable any number of themes and easily switch between them on-demand.</p>

## Preset Themes

Skeleton comes with a curated selection of high quality preset themes out of the box.

<ThemesList />

Tap any of the theme previews above to copy the theme name to your clipboard. Then implement implement the following theme import in your global stylesheet.

```css title="app.css" {3}
/* @import '@skeletonlabs/skeleton'; */
/* @import '@skeletonlabs/skeleton/optional/presets'; */
@import '@skeletonlabs/skeleton/themes/{theme-name}';
```

> Make sure to replace `{theme-name}` with your desired theme names.

## Custom Themes

If you wish to generate your own theme, you can do so using the powerful Skeleton Theme Generator app.

<figure class="linker bg-noise">
	<a href="https://themes.skeleton.dev/" target="_blank" class="btn preset-filled">
		Theme Generator
	</a>
</figure>

1. Open the Theme Generator and customize to your preference.
2. Make sure to set a unique name for your theme.
3. Tap the "code" view from the menu at top-right corner.
4. Tap the "copy" button at the top of copy the theme contents.
5. Create a new file in the root of your project called `theme-custom.ts` (any name is fine).

Open your application's global stylesheet, then append the theme import.

```ts title="tailwind.config" {3}
/* @import '@skeletonlabs/skeleton'; */
/* @import '@skeletonlabs/skeleton/optional/presets'; */
@import '@skeletonlabs/skeleton/themes/{my-theme-name}';
```

## Register Themes

You may register any number of themes by adding addition theme imports to your global stylesheet. Just note that each theme added will slightly increase your projects final CSS bundle size.

```css
@import '@skeletonlabs/skeleton/themes/cerberus';
@import '@skeletonlabs/skeleton/themes/mona';
@import '@skeletonlabs/skeleton/themes/vox';
/* ... */
```

## Activate a Theme

With at least one theme registered, set the active theme using the `data-theme` attribute on your `<html>` element.

```html
<html data-theme="cerberus">
	...
</html>
```

## Customize and Extend

### Modify Properties

If you wish to update or override any Skeleton theme property, even those provided by preset themes, you can do so with the following technique. This will enable you to modify any desired [theme property](/docs/get-started/core-api).

{/* prettier-ignore */}
```css title="app.css"
[data-theme='cerberus'] {
    --spacing: 0.22rem;
    --radius-container: 0.375rem;
    --heading-font-weight: bolder;
}
```

### Target Themes

If your application supports multiple themes, you isolate selection using the `data-theme` attribute.

```css title="app.css"
/** Target Cerberus theme .h1 elements. */
[data-theme='cerberus'] .h1 {
	color: pink;
}
/** Target Mona theme .h1 elements. */
[data-theme='mona'] .h1 {
	color: green;
}
```

### Backgrounds

You can modify your app's background colors in light and dark mode using the following [theme properties](/docs/get-started/core-api#colors).

```css title="app.css"
[data-theme='cerberus'] {
	—body-background-color: 'pink';
	—body-background-color-dark: 'green';
}
```

Background images are also supported, including CSS mesh gradients. By using theme properties for each color, you can ensure the gradient adheres to your theme's palette.

```css title="app.css"
[data-theme='cerberus'] {
	background-image:
		radial-gradient(at 24% 25%, color-mix(in oklab, var(--color-primary-500) 30%, transparent) 0px, transparent 30%),
		radial-gradient(at 35% 13%, color-mix(in oklab, var(--color-success-500) 18%, transparent) 0px, transparent 30%),
		radial-gradient(at 100% 64%, color-mix(in oklab, var(--color-error-500) 3%, transparent) 0px, transparent 40%);
	background-attachment: fixed;
	background-position: center;
	background-repeat: no-repeat;
	background-size: cover;
}
```

We recommend Mesher for generating custom mesh gradients. Colors can be referenced using `var()` and transparency adjusted using `color-mix()`.

### Custom Fonts

Skeleton recommends the use of [Fontsource](https://fontsource.org/) for installing and managing custom fonts.

<figure class="linker bg-noise">
	<a class="btn preset-filled" href="https://fontsource.org/" target="_blank">
		Browse Fontsource
	</a>
</figure>

Install your font of choice.

```console
npm install @fontsource/open-sans
```

Then import each font at the top of your global stylesheet, but below your Tailwind configuration.

```css title="app.css"
import "@fontsource/open-sans";
```

Finally, use the following [theme properties](/docs/get-started/core-api#base-1) to set each respective font-family property. Note that for custom themes, these settings are can be defined directly within each respective theme file.

```css title="app.css"
[data-theme='cerberus'] {
	--heading-font-family: 'Open Sans', sans-serif;
	--base-font-family: 'Open Sans', sans-serif;
	--anchor-font-family: 'inherit';
}
```

It's common to place a single `<Toaster>` in global scope of your app, typically a root layout, this single instance will manage a single toast group with a `toaster` instance (returned by `createToaster`). Any style props set on the `<Toaster>` component are applied to toasts created by the passed in `toaster` instance.

## Promise

Use `toaster.promise()` to create a toast that is able to be configured based on the state of a promise. The toast will automatically update based on the state of the promise.

## Styling Requirement

When using React 18 or below you must manually add the [Required Styles](https://zagjs.com/components/react/toast#requirement).
If you are using React 19 or above, you may skip this step because the styles are automatically applied by the component.

```css title="styles.css"
[data-scope='toast'][data-part='root'] {
	translate: var(--x) var(--y);
	scale: var(--scale);
	z-index: var(--z-index);
	height: var(--height);
	opacity: var(--opacity);
	will-change: translate, opacity, scale;
}

[data-scope='toast'][data-part='root'] {
	transition:
		translate 400ms,
		scale 400ms,
		opacity 400ms;
	transition-timing-function: cubic-bezier(0.21, 1.02, 0.73, 1);
}

[data-scope='toast'][data-part='root'][data-state='closed'] {
	transition:
		translate 400ms,
		scale 400ms,
		opacity 200ms;
	transition-timing-function: cubic-bezier(0.06, 0.71, 0.55, 1);
}
```

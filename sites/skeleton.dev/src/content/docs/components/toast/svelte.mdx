It's common to place a single `<ToastProvider>` in global scope of your app, typically a root layout, this single instance will manage a single toast group with a `toaster` instance (returned by `createToaster`). Any style props set on the `<ToastProvider>` component are applied to toasts created by the passed in `toaster` instance.

## Promise

Use `toaster.promise()` to create a toast that is able to be configured based on the state of a promise. The toast will automatically update based on the state of the promise.

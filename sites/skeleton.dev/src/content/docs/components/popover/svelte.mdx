## Z-Index

By default we do not take an opinionated stance regarding z-index stacking. The result is the component can sometimes be occluded beneath other elements with a higher index. The Z-Index can controlled using the `Positioner` component part. Note that Zag.js opts to style this with a CSS custom property, which can be adjusted using a `style` attribute.

## Headless

Unlike most components in Skeleton, this is provided "headless". This means no default styles are applied out of the box. This ensures you retain full control of all styling.

<Preview client:load>
	<Fragment slot="preview">
		<Styling client:visible />
	</Fragment>
	<Fragment slot="code">
		<Code code={StylingRaw} lang="svelte" />
	</Fragment>
</Preview>

The benefits are as follows:

- You can make the `Trigger` surround any element, including an icon, button, image, etc.
- You can modify the entire structure within `Context`, including custom markup and styling.
- You may place the `CloseTrigger` anywhere, and use it as an X close option.

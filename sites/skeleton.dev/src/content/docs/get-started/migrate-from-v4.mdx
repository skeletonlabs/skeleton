## Prerequisites

We recommend you handle all migration changes on a dedicated feature branch, allowing you to easily revert if needed.

```bash
git checkout -b migration
```

Make sure you've accounted for the following:

- Your app is running the latest release of Skeleton v4.x (`4.15.2`)
- All critical dependencies have been updated (optional but recommended)
- Your app is in a fully functional state before you proceed

Note the `svelte` peer dependency has been raised from `^5.29.0` to `^5.40.0`. React remains unchanged.

```bash
npm install svelte@latest
```

## Themes

### Preset Themes

Preset themes will automatically update with all changes.

### Custom Themes

We recommend importing custom themes into the Theme Generator to automatically convert to the new format. This will also allow you to configure any new or updated settings, while proofing the design.

<figure class="linker bg-noise">
	<a class="btn preset-filled" href="https://themes.skeleton.dev/themes/import" target="_blank" rel="noopener noreferrer">
		Auto-Convert Themes &rarr;
	</a>
</figure>

> NOTE: custom fonts will not be imported. You can implement these via the Typography controls immediately after import.

| v4 Class                | v5 Replacement                                                          | Docs                                                             |
| ----------------------- | ----------------------------------------------------------------------- | ---------------------------------------------------------------- |
| `card-hover`            | These styles are applied automatically to `<a>` and `<button>` elements | [View](/docs/[framework]/tailwind-components/cards#hover)        |
| `input-group`           | `field-group`                                                           | [View](/docs/[framework]/tailwind-components/forms#field-groups) |
| `ig-cell`               | `label label-text`; use a semantic `<label>`                            | --                                                               |
| `ig-input`              | `input`                                                                 | --                                                               |
| `ig-select`             | `select`                                                                | --                                                               |
| `ig-btn`                | `btn`                                                                   | --                                                               |
| `theme-[name]:` variant | Permanently removed                                                     | --                                                               |

### Migrate Field Groups

Input Groups have now been replaced with the much more intuitive [Field Groups](/docs/[framework]/tailwind-components/forms#field-groups) feature.

```html
<!-- v4 -->
<div class="input-group grid-cols-[auto_1fr_auto]">
	<div class="ig-cell preset-tonal">https://</div>
	<input class="ig-input" type="text" />
</div>

<!-- v5 -->
<div class="field-group grid-cols-[auto_1fr_auto]">
	<label class="label label-text preset-tonal" for="url">https://</label>
	<input class="input" type="text" id="url" />
</div>
```

## Brand Color

A new `--color-brand-*` token is now utilized as the default accent color for a number of Skeleton features. Refer to the new [Core APIs](/docs/svelte/get-started/core-api#brand) and updated [Color documentation](/docs/svelte/design/colors#brand-color) details.

- New presets have been added, including: `preset-filled-brand`, `preset-tonal-brand`, `preset-outlined-brand`.
- Brand color is now the default for: checkboxes, radios, circular progress, steps, switches, tabs, and accordions.
- Most Skeleton preset themes default to `primary-500`, which should result in little to no visual changes out of the box.
- The `nouveau` theme is the only exception; it sets a different brand color between light and dark mode.

This can be customized for any preset Skeleton theme (eg: Cerberus) using [this pattern](/docs/svelte/design/themes#modify-properties) in your global stylesheet.

```css
[data-theme='cerberus'] {
	--color-brand-light: var(--color-secondary-500);
	--color-brand-contrast-light: var(--color-secondary-contrast-500);
	--color-brand-dark: var(--color-secondary-500);
	--color-brand-contrast-dark: var(--color-secondary-contrast-500);
}
```

## Element Sizing

### Buttons

Button sizes via `btn-{size}` have been greatly expanded. Refer to [Button Sizes](/docs/svelte/tailwind-components/buttons#sizes) for details.

| v4 Class   | v5 Class   |
| ---------- | ---------- |
| --         | `btn-xs`   |
| `btn-sm`   | `btn-sm`   |
| `btn-base` | `btn-base` |
| `btn-lg`   | `btn-lg`   |
| --         | `btn-xl`   |
| --         | `btn-2xl`  |
| --         | `btn-3xl`  |
| --         | `btn-4xl`  |
| --         | `btn-5xl`  |
| --         | `btn-6xl`  |
| --         | `btn-7xl`  |
| --         | `btn-8xl`  |
| --         | `btn-9xl`  |

### Form Fields

Form fields gained the same scale via new `field-{size}` utility classes. Refer to [Field Sizes](/docs/[framework]/tailwind-components/forms#field-sizes) for details.

### Spacing

A new set of spacing properties using this same scale are available via `spacing-elem-{size}`. Use this for `w-*`, `h-*`, `m-*`, `p-*` and more. Refer to [Element Sizes](/docs/[framework]/get-started/core-api#element-sizes) for details.

```bash
npm run dev
```
